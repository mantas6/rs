# rs

A singleplayer Old School RuneScape–inspired game that runs entirely in the
browser — and entirely headless. The game engine is pure, deterministic
TypeScript driven by 600ms ticks; the React + Canvas UI is just one client
of it. The same public API the UI uses can drive a full play session from a
test, a script, or a bot.

**Live demo:** https://mantas6.github.io/rs/

## Features

- **Lumbridge-style world** (48x40 tiles): a walled castle courtyard with a
  bank, a river with a bridge, a kitchen with a cooking range, a forest, a
  mine, fishing spots, a chicken coop, a fenced cow field, goblins across
  the bridge, and giant rats in the swamp.
- **Skills & XP** — the full OSRS xp curve, levels, boosts/drains, and
  natural stat restore.
- **Gathering** — woodcutting (regular + oak trees), mining (copper, tin,
  iron), and net fishing, with per-tick success rolls and node respawns.
- **Firemaking & cooking** — light fires with a tinderbox, cook on fires or
  ranges, with level-scaled burn chances.
- **Melee combat** — OSRS accuracy/max-hit formulas, attack styles, NPC
  aggression and chasing, drop tables, ground items, and player death.
- **Inventory, equipment & banking** — 28 slots, wieldable tools and
  weapons, and an unlimited stacking bank.
- **Deterministic engine** — a seeded RNG and a manual tick counter mean
  the same seed and commands always replay the exact same game.
- **Synthesized audio** — every sound effect and the music track are
  generated by `scripts/generateAudio.py` (no recorded assets).

## Architecture

```
src/
  engine/            Pure TS game engine (no React/DOM)
    core/            Game loop, tick scheduler, event bus, seeded Rng
    world/           Tile map, pathfinding, ground items
    entities/        Player, NPC entities
    systems/         Skills, combat, gathering, cooking, banking, ...
    setup/           createNewGame(seed): world + starting kit factory
  content/           Data-only definitions: items, npcs, nodes, maps
  ui/                React components + a single <canvas> renderer
```

Three rules keep the design honest:

1. **The engine is pure and deterministic.** No DOM, no `Date.now()`, no
   `Math.random()` — all randomness comes from a seedable `Rng` and all
   timing from the tick counter. The game advances only when `game.tick()`
   is called (the UI calls it every 600ms; tests call it in a loop).
2. **Content is data.** Items, NPCs, resource nodes, and maps are plain
   objects with no logic.
3. **The UI has no game logic.** It issues engine commands
   (`player.walkTo(...)`, `player.attack(...)`) and renders the state.

## Programmatic play

The engine is designed to be played by code. This is a condensed version of
the real end-to-end bot in `src/engine/playthrough.test.ts`, which chops,
burns, fishes, cooks, fights, loots, and banks on the live Lumbridge map:

```ts
import { createNewGame } from './src/engine'

const game = createNewGame(42) // same seed => identical playthrough
const { player } = game

function tickUntil(predicate: () => boolean, maxTicks = 1000): void {
  for (let i = 0; i < maxTicks && !predicate(); i++) game.tick()
}

// Chop a tree until we hold 5 logs.
const tree = game.world.nodes.find((n) => n.def.id === 'tree' && !n.depleted)!
player.gather(tree) // walks there, then chops
tickUntil(() => player.inventory.count('logs') >= 5)

// Light a fire and cook something on it.
player.lightFire('logs')
tickUntil(() => game.fires.fires.length > 0)

// Pick a fight with a goblin.
const goblin = game.npcs.find((n) => n.def.id === 'goblin' && n.alive)!
player.attack(goblin)
tickUntil(() => !goblin.alive)

// Bank the loot.
const booth = game.world.objects.find((o) => o.def.bank)!
player.openBank(booth)
tickUntil(() => game.bank.isOpen)
game.bank.depositAll()
```

## Development

```sh
npm install     # install dependencies
npm run dev     # dev server with the playable UI
npm test        # run the Vitest suite (engine + content + playthrough)
npm run build   # type-check and produce the production build
```

Tests are colocated `*.test.ts` files next to the code under `src/`. The
headless playthrough test (`src/engine/playthrough.test.ts`) plays the full
game loop twice with the same seed and asserts the end states are identical.

Audio is synthesized: run `python3 scripts/generateAudio.py` to regenerate
the `.wav` files in `public/audio`.
